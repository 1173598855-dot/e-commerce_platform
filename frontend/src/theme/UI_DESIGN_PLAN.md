## 一、设计理念

### 1.1 设计原则
- **简洁 (Simplicity)**: 去除冗余元素，聚焦核心功能
- **一致 (Consistency)**: 统一的视觉语言和交互模式
- **易用 (Usability)**: 符合用户习惯，降低学习成本
- **现代 (Modernity)**: 采用当前流行的设计趋势

### 1.2 设计系统
本方案基于**设计令牌 (Design Tokens)** 系统，包括：
- 颜色系统 (Color System)
- 字体系统 (Typography System)
- 间距系统 (Spacing System)
- 阴影系统 (Shadow System)
- 动画系统 (Animation System)

## 三、组件架构

### 3.1 基础组件

#### 按钮 (Button)
- **主按钮**: 橙色背景，白色文字
- **次要按钮**: 白色背景，橙色边框和文字
- **危险按钮**: 红色背景，白色文字
- **小按钮**: 内边距减小
- **禁用状态**: 透明度 60%

#### 输入框 (Input)
- **基础样式**: 白色背景，浅灰边框
- **聚焦状态**: 橙色边框
- **错误状态**: 红色边框
- **禁用状态**: 浅灰背景

#### 卡片 (Card)
- **基础卡片**: 白色背景，小圆角，微小阴影
- **信息卡片**: 用于展示信息
- **选项卡片**: 用于列表选项

#### 标签 (Tag)
- **基础标签**: 橙色背景，白色文字
- **促销标签**: 红色背景，白色文字

### 3.2 导航组件

#### 底部导航 (Tab Bar)

## 五、具体改进方案

### 5.1 首页 (HomeScreen)

#### 当前问题
- 使用 emoji 作为图标，不专业
- 样式基础，缺乏设计感
- 信息层次不清晰

#### 改进方案
1. **搜索栏**
   - 使用 `InputStyles.search` 样式
   - 左侧搜索图标，右侧扫码图标
   - 圆角 20px，浅灰背景

2. **快速导航**
   - 使用 `NavigationStyles.quickNav` 样式
   - 图标容器使用 `NavigationStyles.quickNavIcon` 样式
   - 标签使用 `TextStyles.tag` 样式

3. **分类区域**
   - 使用 `ContainerStyles.section` 样式
   - 标题使用 `TextStyles.h4` 样式
   - 分类网格使用 4 列布局

4. **商品列表**
   - 使用 `CardStyles.product` 样式
   - 两列布局，间距 8px
   - 商品卡片应用新的样式

### 5.2 商品详情页 (ProductDetailScreen)

#### 当前问题
- 图片区域高度固定，不灵活
- 信息区域样式基础
- 底部操作栏样式简单

#### 改进方案
1. **图片区域**
   - 高度改为 350px
   - 背景色改为 `Colors.gray50`
   - 添加图片指示器

2. **信息区域**
   - 使用 `ContainerStyles.info` 样式
   - 价格使用 `TextStyles.price` 样式
   - 标题使用 `TextStyles.h3` 样式
   - 标签使用 `TagStyles.base` 样式

3. **SKU 选择器**
   - 使用 `CardStyles.option` 样式
   - 标签使用 `TextStyles.body` 样式
   - 值使用 `TextStyles.bodySm` 样式

4. **底部操作栏**
   - 使用 `BottomBarStyles.base` 样式
   - 收藏、评论、购物车按钮使用 `BottomBarStyles.actionButton` 样式
   - 立即购买按钮使用 `BottomBarStyles.primaryButton` 样式
   - 加入购物车按钮使用 `BottomBarStyles.secondaryButton` 样式

### 5.3 购物车页 (CartScreen)

#### 当前问题
- 商品卡片样式简单
- 数量选择器样式基础
- 底部结算栏样式简单

#### 改进方案
1. **商品卡片**
   - 使用 `ListStyles.cartItem` 样式
   - 图片区域使用 80x80px，圆角 6px
   - 信息区域使用 `TextStyles.body` 样式

2. **数量选择器**
   - 使用 `ContainerStyles.row` 样式
   - 按钮使用 32x32px，浅灰边框
   - 数量文本使用 `TextStyles.body` 样式

3. **底部结算栏**
   - 使用 `ContainerStyles.footer` 样式
   - 合计文本使用 `TextStyles.body` 样式
   - 合计金额使用 `TextStyles.price` 样式
   - 结算按钮使用 `ButtonStyles.primary` 样式

### 5.4 个人中心页 (ProfileScreen)

#### 当前问题
- 头像样式简单
- 快速入口样式基础
- 菜单项样式不统一

#### 改进方案
1. **用户卡片**
   - 使用 `ContainerStyles.card` 样式
   - 头像使用 64x64px，圆形，橙色背景
   - 用户名使用 `TextStyles.h4` 样式
   - 手机号使用 `TextStyles.bodySm` 样式
   - 编辑按钮使用 `ButtonStyles.secondary` 样式 (小尺寸)

2. **快速入口**
   - 使用 `ContainerStyles.row` 样式
   - 均匀分布，每个入口宽度相等
   - 图标使用 24px
   - 标签使用 `TextStyles.tag` 样式

3. **订单卡片**
   - 使用 `ContainerStyles.card` 样式
   - 标题使用 `TextStyles.h4` 样式
   - 订单标签使用 `NavigationStyles.orderTab` 样式

4. **菜单卡片**
   - 使用 `ContainerStyles.card` 样式
   - 菜单项使用 `ListStyles.item` 样式
   - 图标使用 20px
   - 文本使用 `TextStyles.body` 样式

## 六、可访问性标准

### 6.1 WCAG AA 合规
- **颜色对比度**: 4.5:1 (正常文本), 3:1 (大文本)
- **键盘导航**: 支持 (Web 端)
- **屏幕阅读器**: 支持 (使用语义化标签)
- **焦点管理**: 清晰的焦点指示器

### 6.2 包容性设计
- **触摸目标**: 最小 44px (WCAG 标准)
- **运动敏感性**: 尊重用户偏好 (减少动画)
- **文本缩放**: 支持浏览器文本缩放 up to 200%
- **错误预防**: 清晰的标签、说明和验证
